# Framework Cards — каталог

Разводящая страница для коллекции HTML-карточек фреймворков.
Работает так:

- `index.html` — разводящая страница (сайдбар + главная + вьюер карточек).
- `cards.json` — «оглавление», откуда страница берёт список карточек.
- `*_card.html` — сами карточки (каждая — отдельный самодостаточный HTML).

Страница при загрузке читает `cards.json`, строит сайдбар и сетку на главной. Карточки показываются в iframe, клик по имени в меню = переход на карточку.

## Как опубликовать на GitHub Pages (~5 минут)

### Шаг 1. Регистрация на GitHub

1. Зайдите на [github.com](https://github.com).
2. `Sign up` → почта, пароль, логин (этот логин станет частью адреса сайта). Бесплатный аккаунт.
3. Подтвердите почту.

### Шаг 2. Создание репозитория

1. В правом верхнем углу нажмите `+` → `New repository`.
2. **Repository name** — `framework-cards` (или любое другое, латиницей без пробелов).
3. Оставьте **Public** (бесплатный Pages работает только с публичными репозиториями).
4. Галочки `Add a README`, `Add .gitignore`, `Choose a license` — **не ставьте**, они уже есть в архиве.
5. Нажмите `Create repository`.

### Шаг 3. Загрузка файлов

1. На странице пустого репозитория нажмите `uploading an existing file` (синяя ссылка в центре).
2. Перетащите **все файлы** из архива в область загрузки (включая `index.html`, `cards.json`, `README.md` и все `*_card.html`).
3. Внизу нажмите `Commit changes`.

> Важно: тащите именно **файлы**, не папку целиком. GitHub должен видеть `index.html` в корне репозитория.

### Шаг 4. Включение GitHub Pages

1. В репозитории откройте вкладку `Settings` (верхнее меню).
2. В левой панели выберите `Pages`.
3. В разделе **Source**: выберите `Deploy from a branch`.
4. **Branch**: `main`, папка `/ (root)`. Нажмите `Save`.
5. Подождите ~1 минуту. В верхней части страницы появится зелёная плашка с ссылкой вида:
   `https://ваш-логин.github.io/framework-cards/`
6. Открывайте — каталог должен работать.

## Как добавить новую карточку

1. Откройте репозиторий на github.com.
2. `Add file` → `Upload files` → перетащите файл карточки (например, `SWOT_card.html`).
3. `Commit changes`.
4. Откройте `cards.json` в репозитории, нажмите ✏️ (карандаш — «Edit this file»).
5. Добавьте новый объект в массив `cards` (скопируйте соседний и измените):

```json
{
  "id": "swot",
  "file": "SWOT_card.html",
  "name": "SWOT",
  "subtitle": "Strengths · Weaknesses · Opportunities · Threats",
  "category": "analysis",
  "icon": "📊"
}
```

6. `Commit changes`. Через 30–60 секунд страница обновится, карточка появится в меню.

### Поля карточки

| Поле | Обязательно | Что значит |
|---|---|---|
| `id` | да | Короткий идентификатор на латинице. Используется в URL (`#swot`). |
| `file` | да | Имя HTML-файла карточки в корне репозитория. |
| `name` | да | Название как оно отображается в меню и на главной. |
| `subtitle` | нет | Расшифровка / авторы — показывается под названием. |
| `category` | нет | Ключ категории из блока `categories` (для группировки). |
| `icon` | нет | Эмодзи в списке и на плитке. |

### Как добавить новую категорию

В `cards.json` есть блок `categories` — словарь. Добавьте новый ключ:

```json
"operations": { "name": "Процессы и операции", "icon": "⚙️", "color": "#0f4c54" }
```

И используйте этот ключ в поле `category` у нужных карточек.

## Обновление существующих карточек

1. Если нужно заменить файл — откройте его в репозитории → значок 🗑 → удалите → загрузите новый с тем же именем.
2. Если нужно переименовать карточку — откройте `cards.json` → измените `name`, `subtitle`, `icon` → `Commit changes`.

## Как это работает локально (для отладки)

Прямо дважды кликнуть по `index.html` **не получится** — браузеры блокируют `fetch` для `file://`.

**Вариант 1:** открыть из VS Code с расширением Live Server.

**Вариант 2:** запустить одну команду в терминале из папки каталога:

```bash
python3 -m http.server 8000
```

И открыть `http://localhost:8000`.

## Частые вопросы

**Почему нельзя Google Диск?**
Он отдаёт HTML как «скачиваемый документ», а не как страницу. Браузер не сможет показать карточку в iframe.

**Можно ли сделать репозиторий приватным?**
GitHub Pages для приватных репозиториев доступен только на платном плане (Team/Enterprise). Для публичных карточек обучающего характера обычно достаточно публичного репо.

**Как изменить дизайн разводящей страницы?**
Все стили лежат в `<style>…</style>` внутри `index.html`. Токены на верхнем уровне `:root { … }`.

**Можно ли подключить свой домен?**
Да. В `Settings → Pages → Custom domain` — укажите домен и пропишите у регистратора CNAME на `ваш-логин.github.io`.
