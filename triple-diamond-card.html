01 — Diagram

Три ромба — шесть фаз

Первые два ромба — классическая Double Diamond: проблема и решение. Третий — намеренно не замкнут: отражает реальность итеративной разработки.

02 — Phases

Шесть фаз подробно

Каждый ромб содержит две фазы: расходящуюся (исследование) и сходящуюся (решение). Третий ромб — открытый: итерация не имеет финального конца.

Diamond 1 — Problem Space

Problem Discovery → Problem Definition

Команда исследует проблемное пространство: что болит у пользователей, почему это происходит, какие предположения нужно проверить. Цель — прийти к чёткой формулировке задачи.

Diverge · Problem Discovery

Исследование проблемы

Широкое исследование: интервью, наблюдение, количественный анализ. Цель — увидеть проблему без предвзятостей и собрать максимум контекста.

Converge · Problem Definition

Формулировка задачи

Синтез данных в чёткую постановку. How Might We, Jobs To Be Done, точная формулировка проблемы — до перехода ко второму ромбу.

User InterviewsObservationSurveysAffinity MappingJTBDHow Might WeProblem Statement

Diamond 2 — Solution Space

Solution Discovery → Concept Validation

Второй ромб отвечает на вопрос «как решить?». Команда генерирует широкий спектр идей, затем проверяет концепт на реальных пользователях и принимает решение о разработке.

Diverge · Solution Discovery

Генерация идей

Брейнштормы, Crazy 8, дизайн-спринты. Цель — получить как можно больше концепций до любой фильтрации. Количество важнее качества.

Converge · Concept Validation

Проверка концепта

Прототипирование и тестирование: от paper mockup до кликабельного прототипа. Цель — валидировать ключевые гипотезы до написания кода.

BrainstormingCrazy 8Design SprintWireframesPrototypeUsability TestingA/B Concept

Diamond 3 — Development (Open)

Build & Iterate → Validation → Rollout

Третий ромб намеренно не замкнут: итеративная разработка не имеет финальной точки. Волновая линия внутри символизирует постоянное уточнение. Validation и Rollout — это Early Access и коммерческий запуск.

Build & Iterate

Разработка и итерации

Спринты, инкрементальные релизы, постоянный сбор обратной связи. Каждый цикл уточняет решение на основе реальных данных.

Validation + Rollout

Early Access → Launch

Early Access: ограниченная группа пользователей, финальная валидация. Rollout: поэтапный коммерческий запуск с мониторингом метрик.

03 — Mindset

Расходящееся и сходящееся мышление

Каждый ромб — это чередование двух режимов. Смешивать их нельзя: в фазе расширения нет места критике, в фазе сужения — нет места новым идеям.

Diverge — Расширение

Режим исследования и генерации. Цель — максимально расширить поле возможных ответов, не фильтруя и не оценивая.

Нет «плохих» идей или вопросов
Количество важнее качества
Активное слушание без интерпретации
Откладывать суждения и решения

Converge — Сужение

Режим синтеза и принятия решений. Цель — из широкого поля вариантов выбрать один чёткий путь вперёд.

Критическое мышление и фильтрация
Приоритизация по критериям
Синтез инсайтов в решения
Чёткая точка принятия решения
Ключевое правило

Нельзя перепрыгнуть ромб — каждый отвечает на свой вопрос.

Diamond 1 — правильная проблема. Diamond 2 — правильное решение. Diamond 3 — правильная реализация. Пропуск любого ромба означает строить что-то на непроверенных предположениях.

05 — Mistakes

Типичные ошибки

Большинство проблем — в перескакивании ромбов и смешении режимов Diverge/Converge в одной сессии.

ERR 01

Начинать с решения

Пропустить Diamond 1 и сразу идти в Diamond 2 — значит решать придуманную проблему. Без Problem Definition есть риск построить идеальное решение для несуществующего запроса.

ERR 02

Смешивать Diverge и Converge

Критиковать идеи во время брейншторма или генерировать новые варианты в момент принятия решения — уничтожает оба режима. Они требуют разных установок и разных людей в роли ведущего.

ERR 03

Прототип без пользователей

Concept Validation без реального тестирования — это не валидация. Прототип должен получить обратную связь от тех, для кого он создаётся, иначе это просто артефакт.

ERR 04

Закрытый третий ромб

Попытка «завершить» разработку — категорически неправильное понимание D3. Открытость ромба символизирует, что продукт всегда можно улучшить. Rollout — это не конец.

ERR 05

Фреймворк как жёсткий процесс

Triple Diamond — ориентир, не обязательная последовательность. В реальных проектах ромбы перекрываются, команда может вернуться назад. Это норма, а не ошибка.

ERR 06

Отсутствие точки решения

Без явной «точки схождения» после каждого ромба команда застрянет в бесконечном исследовании или генерации. Каждый ромб должен заканчиваться чётким артефактом-решением.
